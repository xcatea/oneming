# Oneming Knowledge Object Specification

Each knowledge object must follow the same structure.

## Required Fields

- id
- slug
- title
- chineseTitle
- category
- summary
- definition
- explanation
- modernMeaning
- related
- faq
- seoTitle
- seoDescription

## Optional Fields

- aliases
- classicalSource
- useInReports
- examples
- warnings
- schemaType

## Page Structure

Each knowledge page should include:

1. H1
2. One-sentence definition
3. Simple explanation
4. Deeper interpretation
5. How it appears in a Bazi chart
6. Modern meaning
7. Related concepts
8. FAQ
9. Internal links
